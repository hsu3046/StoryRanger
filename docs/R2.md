# Static assets on Cloudflare R2

Images + audio are served from a CDN origin instead of Vercel's bandwidth/image
optimizer. R2 has **zero egress fees**, so media cost scales with storage, not
traffic. `public/` stays the source of truth; R2 is a mirror.

## How it works

- Every media `src` runs through `assetUrl()` ([src/lib/asset-paths.ts](../src/lib/asset-paths.ts)),
  which prefixes root-relative paths (`/stories/…`, `/audio/…`, `/backgrounds/…`,
  `/monsters/…`) with `NEXT_PUBLIC_ASSET_BASE_URL`.
- **Empty base ⇒ same-origin `public/`** (the default — nothing changes in dev).
- **Base set ⇒ served from R2/CDN.** Background/cover images use a plain `<img>`
  (not `next/image`) so they load straight from R2 with no Vercel image-optimizer
  proxy — assets are already pre-optimized to webp.

## One-time Cloudflare setup

1. **Create a bucket** — Cloudflare dashboard → R2 → *Create bucket*
   (e.g. `storyranger-assets`).
2. **Enable public access** (dev): bucket → *Settings* → *Public access* → enable
   the **r2.dev** subdomain. You get a URL like `https://pub-<hash>.r2.dev`.
   - Production: connect a **custom domain** (e.g. `assets.yourdomain.com`)
     instead — r2.dev is rate-limited and not meant for production traffic.
3. **Create S3 API credentials** — R2 → *Manage R2 API Tokens* → create a token
   with **Object Read & Write**. Note the Access Key ID, Secret Access Key, and
   your Account ID.

## Configure env

`.env.local` (never committed):

```bash
# Serving (client) — the bucket's public URL, NO trailing slash:
NEXT_PUBLIC_ASSET_BASE_URL=https://pub-xxxxxxxx.r2.dev

# Upload (server-side script only):
R2_ACCOUNT_ID=xxxxxxxxxxxxxxxx
R2_ACCESS_KEY_ID=xxxxxxxx
R2_SECRET_ACCESS_KEY=xxxxxxxx
R2_BUCKET=storyranger-assets
```

> Leave `NEXT_PUBLIC_ASSET_BASE_URL` empty to keep serving from `public/`.

## Upload / sync media

Needs the AWS CLI (`brew install awscli`). Then:

```bash
./scripts/sync-r2.sh            # upload changed files
./scripts/sync-r2.sh --dry-run  # preview first
```

Re-run it whenever you add or change images/audio. The script reads the `R2_*`
vars from `.env.local`, sets the R2-friendly checksum flags, and
`aws s3 sync ./public → s3://$R2_BUCKET`.

## Verify

1. `./scripts/sync-r2.sh`
2. Set `NEXT_PUBLIC_ASSET_BASE_URL` to the bucket URL, restart `npm run dev`.
3. Open the app → DevTools → Network: image/audio requests should hit the R2
   domain (not `localhost`/`/_next/image`). 404s mean that file wasn't synced.

## Notes

- next/image is removed from the player media path (backgrounds, sprites,
  cover) — direct `<img>` from R2 keeps egress free. The admin sidebar logo
  still uses next/image (local chrome, negligible).
- Newly-dropped files appear locally immediately (dev, empty base) but only on
  R2 after a sync — so author in dev, sync before relying on the R2 base.
- Audio (BGM/SFX) resolves through the same `assetUrl()` in the audio engine.
